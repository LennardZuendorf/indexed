# Connecting Your Data (Configuration)

While you can pass every option via CLI flags, setting up a configuration file makes Indexed much easier to use. It allows you to save source locations, define default behaviors, and securely manage API tokens.

## Interactive Setup

The easiest way to configure Indexed is the interactive mode. This command walks you through setting up or modifying your configuration.

```bash
indexed config update
```

This will prompt you to:
1.  Select a section (e.g., `sources`, `core`).
2.  Choose a setting to change.
3.  Enter a new value.

## Configuration File Structure

Indexed uses a hierarchy of configuration. Settings in `config.toml` are overridden by environment variables, which are in turn overridden by CLI flags.

Your configuration lives in:
-   **Global:** `~/.indexed/config.toml` (default)
-   **Local:** `./.indexed/config.toml` (if you use `--local`)

### Example `config.toml`

```toml
# Core settings for indexing behavior
[core.v1.indexing]
chunk_size = 512
chunk_overlap = 50

# Default search parameters
[core.v1.search]
max_docs = 10

# Source definitions
[sources.jira]
url = "https://your-company.atlassian.net"
email = "user@company.com"
query = "project = PROJ"

[sources.confluence]
url = "https://your-company.atlassian.net/wiki"
read_all_comments = false
```

## Authentication & Security

**Important:** You should never commit your API tokens to version control. Indexed supports `.env` files and environment variables to keep your secrets safe.

### Where to Store Credentials

1.  **Environment Variables:** Set them in your shell session.
2.  **`.env` File:** Create a `.env` file in your `.indexed` directory (global or local).

### Required Variables

| Service | Variable Name | Description |
| :--- | :--- | :--- |
| **Jira / Confluence Cloud** | `ATLASSIAN_TOKEN` | Your API token from id.atlassian.com |
| | `ATLASSIAN_EMAIL` | The email associated with the token |
| **Jira Server** | `JIRA_TOKEN` | Personal Access Token (PAT) |
| **Confluence Server** | `CONF_TOKEN` | Personal Access Token (PAT) |

### How to Get an Atlassian Token (Cloud)
1.  Go to [id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens).
2.  Click **Create API token**.
3.  Label it "Indexed CLI".
4.  Copy the token and paste it into your `.env` file as `ATLASSIAN_TOKEN`.

## Using Configuration vs. CLI Flags

-   **Use Configuration Files (`config.toml`) for:**
    -   Permanent settings (e.g., Jira URL, preferred chunk size).
    -   Default queries you run often.
    -   Project-specific settings (using local mode).

-   **Use CLI Flags for:**
    -   One-off overrides (e.g., `--force` to rebuild).
    -   Testing a new query string.
    -   Pointing to a temporary file path.

You can verify your current active configuration at any time:

```bash
indexed config inspect
```

## Complete Configuration Schema

This section documents all available configuration options in `config.toml`. Configuration uses a hierarchical structure with dot-separated paths.

### Configuration Hierarchy

Configuration values are merged in order of increasing priority:

1. **Pydantic Defaults** (lowest priority)
2. **Global Config** (`~/.indexed/config.toml`)
3. **Workspace Config** (`./.indexed/config.toml`)
4. **Environment Variables** (`INDEXED__section__key`)
5. **CLI Arguments** (highest priority)

### Core Configuration

#### `[core.v1.indexing]`

Indexing pipeline settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `chunk_size` | integer | `512` | Size of text chunks for indexing (1-4096 characters) |
| `chunk_overlap` | integer | `50` | Overlap between consecutive chunks (0 to chunk_size-1) |
| `batch_size` | integer | `32` | Batch size for processing documents (≥1) |

**Example:**
```toml
[core.v1.indexing]
chunk_size = 1024
chunk_overlap = 100
batch_size = 64
```

**Note:** `chunk_overlap` must be less than `chunk_size`.

#### `[core.v1.embedding]`

Embedding generation settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `provider` | string | `"sentence-transformers"` | Embedding provider (sentence-transformers, openai, voyage) |
| `model_name` | string | `"all-MiniLM-L6-v2"` | Model name for embeddings |
| `dimension` | integer | `null` | Embedding dimension (auto-detected if null) |
| `batch_size` | integer | `64` | Batch size for embedding generation (≥1) |
| `device` | string | `null` | Device for embeddings (cpu, cuda, mps, or null for auto) |
| `api_key_env` | string | `null` | Environment variable name for API key (if needed) |

**Example:**
```toml
[core.v1.embedding]
provider = "sentence-transformers"
model_name = "all-MiniLM-L6-v2"
batch_size = 64
```

#### `[core.v1.storage]`

Vector storage settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `type` | string | `"faiss"` | Storage backend type |
| `index_type` | string | `"IndexFlatL2"` | FAISS index type (IndexFlatL2, IndexIVFFlat, IndexHNSW) |
| `persistence_enabled` | boolean | `true` | Enable persistence to disk |
| `persistence_path` | string | `null` | Path for persisting indexes (defaults to `.indexed/v1/storage`) |

**Example:**
```toml
[core.v1.storage]
type = "faiss"
index_type = "IndexFlatL2"
persistence_enabled = true
```

#### `[core.v1.search]`

Search behavior settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `max_docs` | integer | `10` | Maximum documents to return (1-100) |
| `max_chunks` | integer | `30` | Maximum chunks to return (≥1) |
| `include_full_text` | boolean | `false` | Include full document text in results |
| `include_all_chunks` | boolean | `false` | Include all document chunks |
| `include_matched_chunks` | boolean | `false` | Include only matched chunks |
| `score_threshold` | float | `null` | Minimum similarity score threshold (0.0-1.0) |

**Example:**
```toml
[core.v1.search]
max_docs = 10
max_chunks = 30
score_threshold = 0.7
```

### Source Configuration

#### `[sources.jira]`

Jira configuration (works for both Cloud and Server/Data Center).

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `url` | string | Yes | Jira instance URL (e.g., `https://company.atlassian.net` or `https://jira.company.com`) |
| `query` | string | Yes | JQL (Jira Query Language) query string |
| `email` | string | No* | Atlassian account email (Cloud only, env: `ATLASSIAN_EMAIL`) |
| `api_token` | string | No* | API token (Cloud only, env: `ATLASSIAN_TOKEN`) |
| `token` | string | No* | Bearer token (Server only, env: `JIRA_TOKEN`) |
| `login` | string | No* | Username (Server only, env: `JIRA_LOGIN`) |
| `password` | string | No* | Password (Server only, env: `JIRA_PASSWORD`) |

\* Authentication is required but should be stored in `.env` file, not `config.toml`.

**Example (Cloud):**
```toml
[sources.jira]
url = "https://company.atlassian.net"
query = "project = PROJ AND priority = High"
email = "user@company.com"  # Or use ATLASSIAN_EMAIL in .env
```

**Example (Server):**
```toml
[sources.jira]
url = "https://jira.company.com"
query = "project = PROJ"
# Use JIRA_TOKEN, JIRA_LOGIN, or JIRA_PASSWORD in .env
```

#### `[sources.confluence]`

Confluence configuration (works for both Cloud and Server/Data Center).

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `url` | string | Yes | Confluence instance URL (e.g., `https://company.atlassian.net/wiki` or `https://confluence.company.com`) |
| `query` | string | Yes | CQL (Confluence Query Language) query string |
| `read_all_comments` | boolean | No | Read nested comments (default: `true`) |
| `email` | string | No* | Atlassian account email (Cloud only, env: `ATLASSIAN_EMAIL`) |
| `api_token` | string | No* | API token (Cloud only, env: `ATLASSIAN_TOKEN`) |
| `token` | string | No* | Bearer token (Server only, env: `CONF_TOKEN`) |
| `login` | string | No* | Username (Server only, env: `CONF_LOGIN`) |
| `password` | string | No* | Password (Server only, env: `CONF_PASSWORD`) |

\* Authentication is required but should be stored in `.env` file, not `config.toml`.

**Example (Cloud):**
```toml
[sources.confluence]
url = "https://company.atlassian.net/wiki"
query = "space = ENG AND type = page"
read_all_comments = true
```

**Example (Server):**
```toml
[sources.confluence]
url = "https://confluence.company.com"
query = "space = DOCS"
read_all_comments = false
# Use CONF_TOKEN, CONF_LOGIN, or CONF_PASSWORD in .env
```

#### `[sources.files]`

Local files configuration.

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `path` | string | Yes | Path to files or directory (must exist) |
| `include_patterns` | array of strings | No | Regex patterns for files to include (default: `[".*"]`) |
| `exclude_patterns` | array of strings | No | Regex patterns for files/directories to exclude (default: `[]`) |
| `fail_fast` | boolean | No | Stop indexing on first error (default: `false`) |

**Example:**
```toml
[sources.files]
path = "/path/to/documents"
include_patterns = [".*\\.md$", ".*\\.txt$"]
exclude_patterns = ["node_modules/.*", ".*\\.log$"]
fail_fast = false
```

### Storage Configuration

#### `[storage]`

Storage mode settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `mode` | string | `"global"` | Storage mode: `"global"` (`~/.indexed/`) or `"local"` (`./.indexed/`) |

**Example:**
```toml
[storage]
mode = "local"  # Use project-specific storage
```

### Logging Configuration

#### `[logging]`

Global logging settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `level` | string | `"WARNING"` | Log level: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` |
| `as_json` | boolean | `false` | Emit JSON-formatted logs |

**Example:**
```toml
[logging]
level = "INFO"
as_json = false
```

### MCP Configuration

#### `[mcp]`

Model Context Protocol server settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `host` | string | `"localhost"` | MCP server host |
| `port` | integer | `8000` | MCP server port (1-65535) |
| `log_level` | string | `"WARNING"` | MCP server log level |
| `enable_async_pool` | boolean | `false` | Enable async processing pool |
| `include_index_size` | boolean | `false` | Include index size in MCP responses |

**Example:**
```toml
[mcp]
host = "localhost"
port = 8000
log_level = "INFO"
```

### Performance Configuration

#### `[performance]`

Performance and caching settings.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enable_cache` | boolean | `true` | Enable search result caching |
| `cache_max_entries` | integer | `32` | Maximum cache entries (≥1) |
| `log_sqlite_queries` | boolean | `false` | Log SQLite queries for debugging |

**Example:**
```toml
[performance]
enable_cache = true
cache_max_entries = 64
```

### Paths Configuration

#### `[paths]`

File system path settings (advanced).

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `collections_dir` | string | (auto) | Directory for collections storage |
| `caches_dir` | string | (auto) | Directory for document caches |
| `temp_dir` | string | `"./tmp"` | Temporary files directory |

**Note:** These paths default to the storage location (`~/.indexed/data/...` or `./.indexed/data/...`) unless explicitly configured.

## Environment Variables

You can override any configuration value using environment variables with the format:

```
INDEXED__section__subsection__key=value
```

Use double underscores (`__`) to separate sections and keys.

### Examples

```bash
# Set chunk size
export INDEXED__core__v1__indexing__chunk_size=1024

# Set Jira URL
export INDEXED__sources__jira__url=https://company.atlassian.net

# Set storage mode
export INDEXED__storage__mode=local

# Set log level
export INDEXED__logging__level=DEBUG
```

### Complete Example Configuration

Here's a complete example `config.toml`:

```toml
# Core indexing settings
[core.v1.indexing]
chunk_size = 512
chunk_overlap = 50
batch_size = 32

# Search settings
[core.v1.search]
max_docs = 10
max_chunks = 30
score_threshold = 0.7

# Jira source (Cloud)
[sources.jira]
url = "https://company.atlassian.net"
query = "project = PROJ AND updated > -30d"
# email and api_token in .env file

# Confluence source (Cloud)
[sources.confluence]
url = "https://company.atlassian.net/wiki"
query = "space = ENG AND type = page"
read_all_comments = true

# Files source
[sources.files]
path = "/path/to/documents"
include_patterns = [".*\\.md$", ".*\\.txt$"]
exclude_patterns = ["node_modules/.*"]

# Storage mode
[storage]
mode = "global"

# Logging
[logging]
level = "WARNING"
as_json = false

# MCP server
[mcp]
host = "localhost"
port = 8000
log_level = "WARNING"
```

**Remember:** Never commit API tokens or passwords to `config.toml`. Store them in `.env` files instead.
