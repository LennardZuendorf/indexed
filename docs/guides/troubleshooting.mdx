# Troubleshooting & Maintenance

This guide helps you keep your collections healthy and fix common problems.

## Inspecting Collections

If you're unsure what data is in your index, use the inspect command.

```bash
indexed index inspect
```

This will show:
-   A list of all collections.
-   The number of documents in each.
-   When they were last updated.
-   The source configuration.

## Common Tasks

### 1. Updating a Collection
Data sources change. New files are added, Jira tickets are closed. To keep your index in sync:

```bash
indexed index update
# Or update a specific one:
indexed index update --collection my-docs
```

### 2. Removing a Collection
To delete an index completely (this frees up disk space):

```bash
indexed index remove --collection old-project
```

### 3. Re-indexing from Scratch
If you suspect an index is corrupted or want to change chunking settings, force a rebuild:

```bash
indexed index create files --name my-docs --path ./docs --force
```

## Troubleshooting Common Issues

### Authentication Issues

#### "Authentication Failed" or "401 Unauthorized"

**Symptoms:** Indexing fails with authentication errors when connecting to Jira or Confluence.

**Common Causes:**
- Invalid or expired API token
- Email address doesn't match the token owner
- Token has insufficient permissions
- Extra spaces or newlines in token string

**Solutions:**
1. **Verify token in `.env` file:**
   ```bash
   # Check your .env file location
   cat ~/.indexed/.env  # Global
   cat ./.indexed/.env   # Local
   ```

2. **For Jira/Confluence Cloud:**
   - Ensure `ATLASSIAN_EMAIL` matches the account that generated the token
   - Regenerate token at [id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
   - Copy token immediately (it's only shown once)

3. **Check for formatting issues:**
   - Remove any leading/trailing spaces
   - Ensure no line breaks in the token
   - Use quotes in `.env` if token contains special characters

4. **Verify permissions:**
   - Token must have read access to the projects/spaces you're indexing
   - For Confluence, ensure the user has space permissions

**Prevention:** Store tokens in `.env` files (never commit to version control) and rotate tokens periodically.

#### "Token expired or invalid"

**Symptoms:** Previously working authentication suddenly fails.

**Solutions:**
1. **Regenerate API token** (tokens can expire or be revoked)
2. **Check token age:** Some organizations have token expiration policies
3. **Verify account status:** Ensure the Atlassian account is active
4. **Update `.env` file** with the new token

#### "Permission denied errors"

**Symptoms:** Authentication succeeds but indexing fails with permission errors.

**Solutions:**
1. **Check project/space permissions:**
   - Verify the API token user has read access
   - For Jira: Check project permissions in Jira settings
   - For Confluence: Check space permissions

2. **Test with a simpler query:**
   ```bash
   # Try a broader query first
   indexed index create jira --query "project = PROJ"
   ```

3. **Use admin account:** If possible, use an account with broader permissions for indexing

### Indexing Issues

#### "No documents found"

**Symptoms:** Indexing completes but reports 0 documents indexed.

**Common Causes:**
- File patterns too restrictive
- Query returns no results
- Permission issues
- Incorrect paths

**Solutions:**

1. **For file indexing:**
   ```bash
   # Check include patterns (case-sensitive on Linux/macOS)
   indexed index create files --path ./docs --include "*.md" --include "*.txt"

   # Use verbose mode to see what's being scanned
   indexed --verbose index create files --path ./docs
   ```

2. **For Jira:**
   - Test JQL query in Jira web interface first
   - Start with a simple query: `project = PROJ`
   - Verify the project key is correct (case-sensitive)

3. **For Confluence:**
   - Check space key spelling
   - Verify CQL syntax: `space = SPACEKEY AND type = page`
   - Ensure user has read access to the space

4. **Check file paths:**
   - Use absolute paths if relative paths don't work
   - Verify the directory exists and is readable
   - Check for symlink issues

**Prevention:** Always test queries/patterns in the source system first, use `--verbose` to see what's being processed.

#### "Indexing is very slow"

**Symptoms:** Indexing takes much longer than expected.

**Common Causes:**
- Large documents or many files
- Slow network connection (for Jira/Confluence)
- Insufficient system resources
- Complex file formats (PDFs, large DOCX files)

**Solutions:**

1. **Check system resources:**
   ```bash
   # Monitor CPU and memory usage
   top  # Linux/macOS
   # or
   htop
   ```

2. **Optimize indexing:**
   - Index in smaller batches (create multiple collections)
   - Use more specific queries to reduce document count
   - Exclude unnecessary file types: `--exclude "*.log" --exclude "*.tmp"`

3. **For network sources (Jira/Confluence):**
   - Check network connection speed
   - Index during off-peak hours
   - Use more specific queries to reduce API calls

4. **Adjust chunking settings:**
   ```bash
   # Smaller chunks may process faster
   indexed config set core.v1.indexing.chunk_size 256
   ```

**Prevention:** Start with smaller collections to establish baseline performance, then scale up.

#### "Out of memory errors during indexing"

**Symptoms:** Indexing crashes with memory errors, especially with large collections.

**Solutions:**

1. **Reduce batch size:**
   - Create smaller collections
   - Index in chunks (e.g., by date range for Jira)

2. **Free up system memory:**
   - Close other applications
   - Increase available RAM if possible

3. **Optimize chunking:**
   ```bash
   # Smaller chunks use less memory
   indexed config set core.v1.indexing.chunk_size 256
   ```

4. **Index incrementally:**
   - Use `index update` instead of recreating large collections
   - Index new documents separately, then merge

**Prevention:** Monitor memory usage during first indexing run, adjust collection size accordingly.

#### "Some files are skipped"

**Symptoms:** Not all expected files are indexed.

**Solutions:**

1. **Check file format support:**
   - Verify file extension is supported (see [FAQ](./faq.mdx))
   - Unsupported formats are automatically skipped

2. **Review include/exclude patterns:**
   ```bash
   # List what would be indexed (dry run)
   indexed --verbose index create files --path ./docs
   ```

3. **Check file permissions:**
   - Ensure files are readable
   - Check for permission errors in verbose output

4. **Verify file encoding:**
   - Some binary or corrupted files may be skipped
   - Check logs for specific error messages

**Prevention:** Use `--verbose` to see which files are processed and which are skipped.

### Search Issues

#### "Search returns no results"

**Symptoms:** Search completes but shows no results.

**Solutions:**

1. **Verify collection exists:**
   ```bash
   indexed index inspect
   ```

2. **Check collection has documents:**
   - Ensure indexing completed successfully
   - Verify document count in inspect output

3. **Try different queries:**
   - Start with simple, common terms
   - Use more specific queries if collection is large
   - Try synonyms or related terms

4. **Check collection name:**
   ```bash
   # If using --collection, verify name matches exactly
   indexed index search "query" --collection exact-name
   ```

5. **Rebuild index if corrupted:**
   ```bash
   indexed index remove --collection my-collection
   indexed index create files --name my-collection --path ./docs
   ```

**Prevention:** Always verify indexing completed successfully before searching.

#### "Search results are not relevant"

**Symptoms:** Search returns results but they don't match the query intent.

**Solutions:**

1. **Refine your query:**
   - Use more specific terms
   - Include context: "authentication token expiration" vs "token"
   - Try natural language questions

2. **Check what was indexed:**
   ```bash
   indexed index inspect --collection my-collection
   ```

3. **Adjust chunking settings:**
   ```bash
   # Larger chunks may provide better context
   indexed config set core.v1.indexing.chunk_size 1024
   ```

4. **Re-index with better settings:**
   - Remove and recreate collection with optimized settings
   - Ensure source documents are well-structured

**Prevention:** Use descriptive document titles and headings, maintain consistent terminology in source documents.

#### "Search is slow"

**Symptoms:** Search queries take a long time to complete.

**Solutions:**

1. **Limit results:**
   ```bash
   indexed index search "query" --limit 5
   ```

2. **Search specific collection:**
   ```bash
   # Faster than searching all collections
   indexed index search "query" --collection my-collection
   ```

3. **Check collection size:**
   - Very large collections (>100K documents) may be slower
   - Consider splitting into multiple smaller collections

4. **Verify system resources:**
   - Ensure sufficient RAM available
   - Close other resource-intensive applications

**Prevention:** Keep collections at reasonable sizes (<50K documents per collection).

### Configuration Issues

#### "Configuration not found"

**Symptoms:** Indexed can't find or load configuration.

**Solutions:**

1. **Check active configuration:**
   ```bash
   indexed config inspect
   ```

2. **Verify storage mode:**
   ```bash
   # Check if using local or global mode
   indexed config inspect --json | grep mode
   ```

3. **Use explicit flags:**
   ```bash
   # Force local mode
   indexed --local config inspect

   # Force global mode
   indexed --global config inspect
   ```

4. **Reinitialize configuration:**
   ```bash
   indexed config init
   ```

**Prevention:** Be consistent with `--local` or `--global` flags, or set `storage.mode` in config.

#### "Invalid configuration format"

**Symptoms:** Configuration errors when starting Indexed.

**Solutions:**

1. **Validate configuration:**
   ```bash
   indexed config validate
   ```

2. **Check TOML syntax:**
   - Ensure proper section headers: `[section.subsection]`
   - Verify no syntax errors (missing brackets, quotes, etc.)
   - Use a TOML validator online

3. **Check for type errors:**
   - Numbers should be integers, not strings
   - Booleans should be `true`/`false`, not `"true"`/`"false"`

4. **Reset to defaults:**
   ```bash
   # Backup first, then reset
   cp ~/.indexed/config.toml ~/.indexed/config.toml.backup
   rm ~/.indexed/config.toml
   indexed config init
   ```

**Prevention:** Use `indexed config set` command instead of editing TOML directly, or validate after manual edits.

#### "Environment variables not working"

**Symptoms:** Environment variables don't override config values.

**Solutions:**

1. **Check variable format:**
   ```bash
   # Correct format: INDEXED__section__key
   export INDEXED__core__v1__indexing__chunk_size=1024

   # Wrong: INDEXED_core_v1_indexing_chunk_size
   ```

2. **Verify variable is set:**
   ```bash
   echo $INDEXED__core__v1__indexing__chunk_size
   ```

3. **Check variable name:**
   - Use double underscores `__` between sections
   - Section names should match config structure exactly

4. **Load in same shell:**
   - Export variables in the same terminal session
   - Or use `.env` file instead

**Prevention:** Use `.env` files for persistent environment variables instead of shell exports.

### Storage Issues

#### "Disk space running low"

**Symptoms:** Warnings about disk space or indexing fails due to space.

**Solutions:**

1. **Check disk usage:**
   ```bash
   # Check Indexed data directory size
   du -sh ~/.indexed/data/collections/
   ```

2. **Remove unused collections:**
   ```bash
   indexed index remove --collection old-collection
   ```

3. **Clean up old indexes:**
   - Remove collections you no longer need
   - Archive collections to external storage

4. **Optimize collection size:**
   - Use more specific queries to reduce indexed documents
   - Exclude large binary files that don't need indexing

**Prevention:** Monitor disk usage regularly, set up alerts if possible.

#### "Collection corrupted"

**Symptoms:** Collection exists but can't be searched or inspected.

**Solutions:**

1. **Inspect collection:**
   ```bash
   indexed index inspect --collection problematic-collection
   ```

2. **Remove and recreate:**
   ```bash
   indexed index remove --collection problematic-collection
   indexed index create files --name problematic-collection --path /original/path
   ```

3. **Check file permissions:**
   ```bash
   ls -la ~/.indexed/data/collections/problematic-collection/
   ```

4. **Verify disk integrity:**
   - Check for disk errors
   - Ensure sufficient disk space

**Prevention:** Regular backups, avoid interrupting indexing process, ensure stable power/system.

#### "Cannot access collection"

**Symptoms:** Permission errors when accessing collections.

**Solutions:**

1. **Check file permissions:**
   ```bash
   ls -la ~/.indexed/data/collections/
   ```

2. **Fix permissions:**
   ```bash
   # Make readable/writable by user
   chmod -R u+rw ~/.indexed/data/collections/
   ```

3. **Check ownership:**
   ```bash
   # If wrong owner, fix it
   sudo chown -R $USER ~/.indexed/
   ```

4. **Verify storage location:**
   - Ensure you're using the correct storage mode (local vs global)
   - Check if collection exists in expected location

**Prevention:** Ensure consistent user account, avoid using sudo for Indexed operations.

### MCP Issues

#### "MCP server won't start"

**Symptoms:** `indexed mcp run` fails or doesn't start.

**Solutions:**

1. **Check for errors:**
   ```bash
   indexed --verbose mcp run
   ```

2. **Verify configuration:**
   ```bash
   indexed config inspect mcp
   ```

3. **Test in dev mode:**
   ```bash
   indexed mcp dev
   ```

4. **Check port availability (for HTTP/SSE):**
   ```bash
   # If using HTTP transport, ensure port is free
   lsof -i :8000  # Check if port 8000 is in use
   ```

5. **Verify Python environment:**
   - Ensure Indexed is properly installed
   - Check Python version (3.11+)

**Prevention:** Test MCP server with `mcp dev` before integrating with AI tools.

#### "AI agent can't connect"

**Symptoms:** Claude Desktop or other AI tools can't connect to Indexed MCP server.

**Solutions:**

1. **Verify MCP server is running:**
   ```bash
   indexed mcp inspect
   ```

2. **Check Claude Desktop configuration:**
   - Verify JSON syntax is correct
   - Ensure command path is correct: `uv tool run indexed-cli mcp run`
   - Check Claude Desktop logs for errors

3. **Test command manually:**
   ```bash
   # Run the exact command from Claude Desktop config
   uv tool run indexed-cli mcp run
   ```

4. **Check transport mode:**
   - Claude Desktop uses `stdio` by default
   - For HTTP/SSE, ensure correct host/port in config

5. **Restart Claude Desktop:**
   - Configuration changes require restart
   - Check for updates to Claude Desktop

**Prevention:** Use `mcp dev` to test server before configuring AI tools, verify configuration syntax.

#### "MCP tools not available"

**Symptoms:** MCP server connects but tools don't appear in AI agent.

**Solutions:**

1. **Inspect available tools:**
   ```bash
   indexed mcp inspect
   ```

2. **Verify collections exist:**
   ```bash
   indexed index inspect
   ```

3. **Check MCP server version:**
   - Ensure Indexed version supports MCP
   - Update Indexed if needed

4. **Review AI tool logs:**
   - Check Claude Desktop logs for MCP errors
   - Look for tool registration failures

5. **Test tools directly:**
   ```bash
   # Use MCP inspector to test tools
   indexed mcp dev
   ```

**Prevention:** Keep Indexed updated, verify MCP compatibility with AI tool version.

### Migration from Legacy Versions
If you used an older version of Indexed, your data might be in `./data` instead of `~/.indexed`.
Run the migration tool to move it:

```bash
indexed config migrate
```

## Logs and Debugging

If you are stuck, you can enable verbose logging to see exactly what is happening under the hood.

```bash
# Enable verbose logging
indexed --verbose index create ...

# Output logs as JSON (useful for parsing)
indexed --json-logs index create ...
```
