# Getting Started with Indexed

Indexed helps you create searchable collections from your documents—whether they live in local files, Jira, or Confluence—and makes them accessible to you and your AI agents.

This guide will get you from zero to your first searchable index in under 5 minutes.

## Prerequisites

Before installing Indexed, ensure you have the following:

### Python Version

Indexed requires **Python 3.11 or higher**. Check your Python version:

```bash
python --version
# or
python3 --version
```

If you need to upgrade Python, visit [python.org](https://www.python.org/downloads/) or use a version manager like `pyenv`.

### Operating System

Indexed runs on:
- **Linux** (most distributions)
- **macOS** (10.14+)
- **Windows** (10+)

### System Requirements

- **RAM:** At least 4GB available (8GB+ recommended for large collections)
- **Disk Space:**
  - Base installation: ~500MB
  - Per collection: ~10-50MB per 1,000 documents (varies by document size)
  - Plan for 2-3x your source data size for indexes and metadata
- **Internet:** Required only for initial installation and downloading embedding models (first run)

### Package Manager (Optional)

While not required, we recommend using `uv` for faster, more reliable package management. If you prefer `pip`, that works too.

## 1. Installation

Indexed is a Python application. You can install it using `uv` (recommended) or `pip`.

### Option 1: Using uv (Recommended)

`uv` is a fast Python package installer. If you don't have it installed:

```bash
# Install uv
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Then install Indexed:

```bash
uv tool install indexed-cli
```

### Option 2: Using pip

If you prefer `pip`, you can install Indexed directly:

```bash
# Standard installation
pip install indexed-cli

# Or with a virtual environment (recommended)
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install indexed-cli
```

### Option 3: From Source

For development or to get the latest features:

```bash
git clone https://github.com/your-org/indexed.git
cd indexed
uv sync --all-groups
```

### Verify Installation

After installation, verify it works:

```bash
indexed --version
```

You should see the version number. If you get a "command not found" error:

- **On Linux/macOS:** Ensure `~/.local/bin` is in your PATH (add to `~/.bashrc` or `~/.zshrc`):
  ```bash
  export PATH="$HOME/.local/bin:$PATH"
  ```
- **On Windows:** Ensure Python Scripts directory is in your PATH
- **With uv:** Run `uv tool run indexed-cli --version` instead

### Troubleshooting Installation

**Python version errors:**
- Ensure Python 3.11+ is installed and accessible
- Use `python3` instead of `python` if needed

**Permission errors:**
- Use `pip install --user indexed-cli` to install in user directory
- Or use a virtual environment to avoid permission issues

**Missing dependencies:**
- Indexed will install required dependencies automatically
- If you see errors, ensure you have a working internet connection
- On some systems, you may need: `pip install --upgrade pip setuptools wheel`

## 2. Initialize Your Workspace

Before you start indexing, initialize a configuration workspace. This creates a local `.indexed` directory to store your settings and data.

```bash
indexed config init
```

This command creates:
- `config.toml`: Your configuration file.
- `.env`: A place for API tokens (if you use Jira/Confluence).
- `data/`: The directory where your indexes will live.

## 3. Create Your First Collection

Let's start by indexing some local files. A "Collection" is just a named bucket of searchable documents.

Suppose you have a folder of notes or documentation at `./docs`. Run:

```bash
indexed index create files --name my-docs --path ./docs
```

What happens next:
1.  **Reading:** Indexed scans your folder for supported files (Markdown, Text, PDF, etc.).
2.  **Chunking:** It breaks documents into smaller, meaningful pieces.
3.  **Embedding:** It converts text into vector representations locally (no data leaves your machine).
4.  **Indexing:** It saves a searchable index to disk.

## 4. Search Your Collection

Now that your documents are indexed, you can search them using natural language.

```bash
indexed index search "how do I configure authentication"
```

You'll see a list of results ranked by relevance, showing the source file and the matching text context.

## Core Concept: Collections

In Indexed, everything revolves around **Collections**.

-   A Collection is a source-specific index (e.g., "Engineering Docs" from Confluence, or "Q4 Plans" from local files).
-   You can search across one specific collection or all of them at once.
-   Your data stays local. Indexed is designed for privacy and offline capability.
