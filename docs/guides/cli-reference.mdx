# CLI Command Reference

This is a comprehensive reference for all Indexed CLI commands. Indexed groups commands by resource: **Index** (collections), **Config** (settings), and **MCP** (AI integration).

> **Tip:** For quick help on any command, use `--help` (e.g., `indexed index create --help`).

---

## Index Commands

Commands for managing your searchable collections.

### `index create files`

Create a collection from local files and directories.

**Syntax:**
```bash
indexed index create files [OPTIONS]
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--collection` | `-c` | Name of the collection | `files` |
| `--path` | `-p` | Path to root directory or file(s) | (from config or prompt) |
| `--include` | | Regex pattern for files to include (can repeat) | (all files) |
| `--exclude` | | Regex pattern for files/directories to exclude (can repeat) | (none) |
| `--fail-fast` | | Stop on first file read error | `false` |
| `--use-cache` | | Enable on-disk cache for faster reindexing | `true` |
| `--no-cache` | | Disable caching | (opposite of `--use-cache`) |
| `--force` | | Delete existing collection with same name before creating | `false` |
| `--verbose` | `-v` | Enable verbose (INFO) logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level (DEBUG, INFO, WARNING, ERROR) | (default) |

**Examples:**

```bash
# Basic: Create collection from directory
indexed index create files --name my-docs --path ./documents

# With include/exclude patterns
indexed index create files \
  --name project-docs \
  --path ~/projects/alpha/docs \
  --include "*.md" \
  --include "*.txt" \
  --exclude "node_modules/*" \
  --exclude "_site/*"

# Force rebuild (delete existing first)
indexed index create files --name my-docs --path ./docs --force

# Disable caching for fresh index
indexed index create files --name my-docs --path ./docs --no-cache

# Verbose output to see progress
indexed index create files --name my-docs --path ./docs --verbose
```

**Output:** Shows progress bar during indexing, then summary of documents indexed.

### `index create jira`

Create a collection from Jira issues.

**Syntax:**
```bash
indexed index create jira [OPTIONS]
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--collection` | `-c` | Name of the collection | `jira` |
| `--url` | | Jira instance URL (Cloud or Server) | (from config) |
| `--query` | | JQL (Jira Query Language) query string | (from config) |
| `--email` | | Email for authentication (Cloud only) | (from config) |
| `--force` | | Delete existing collection before creating | `false` |
| `--verbose` | `-v` | Enable verbose logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level | (default) |

**Examples:**

```bash
# Create from Jira Cloud
indexed index create jira \
  --name urgent-bugs \
  --url https://company.atlassian.net \
  --query "project = BACKEND AND priority = High AND updated > -30d"

# Create from Jira Server
indexed index create jira \
  --name all-issues \
  --url https://jira.company.com \
  --query "project = PROJ"

# Using config values (just specify collection name)
indexed index create jira --name my-jira
```

**Authentication:** Requires API token in `.env` file:
- Cloud: `ATLASSIAN_TOKEN` and `ATLASSIAN_EMAIL`
- Server: `JIRA_TOKEN`

### `index create confluence`

Create a collection from Confluence pages.

**Syntax:**
```bash
indexed index create confluence [OPTIONS]
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--collection` | `-c` | Name of the collection | `confluence` |
| `--url` | | Confluence instance URL | (from config) |
| `--query` | | CQL (Confluence Query Language) query | (from config) |
| `--read-all-comments` | | Include all comments in indexing | `true` |
| `--no-read-all-comments` | | Skip comments | (opposite) |
| `--email` | | Email for authentication (Cloud only) | (from config) |
| `--force` | | Delete existing collection before creating | `false` |
| `--verbose` | `-v` | Enable verbose logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level | (default) |

**Examples:**

```bash
# Index Confluence space
indexed index create confluence \
  --name engineering-wiki \
  --url https://company.atlassian.net/wiki \
  --query "space = ENG AND type = page"

# Skip comments for faster indexing
indexed index create confluence \
  --name docs \
  --query "space = DOCS" \
  --no-read-all-comments
```

**Authentication:** Same as Jira (see above).

### `index search`

Search across one or all collections using semantic similarity.

**Syntax:**
```bash
indexed index search <QUERY> [OPTIONS]
```

**Arguments:**

- `QUERY` (required): Search query in natural language

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--collection` | `-c` | Collection name to search (omit for all) | (all collections) |
| `--limit` | `-l` | Number of results per collection | `5` |
| `--compact` | | Show compact list instead of cards | `false` |
| `--no-content` | | Hide content previews | `false` |
| `--verbose` | `-v` | Enable verbose logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level | (default) |

**Examples:**

```bash
# Search all collections
indexed index search "how do I configure authentication"

# Search specific collection
indexed index search "deployment process" --collection my-docs

# Limit results
indexed index search "bug fix" --limit 3

# Compact view
indexed index search "API documentation" --compact

# Hide content previews
indexed index search "error handling" --no-content

# Combine options
indexed index search "deployment" --collection prod-docs --limit 10 --compact
```

**Output:** Results shown as cards (or compact list) with relevance scores, source, and content preview.

### `index update`

Refresh a collection with the latest data from its source.

**Syntax:**
```bash
indexed index update [COLLECTION] [OPTIONS]
```

**Arguments:**

- `COLLECTION` (optional): Collection name to update (omit to update all)

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--verbose` | `-v` | Enable verbose logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level | (default) |

**Examples:**

```bash
# Update specific collection
indexed index update my-docs

# Update all collections
indexed index update

# Verbose output
indexed index update my-docs --verbose
```

**What it does:**
- Re-scans source for new documents
- Updates modified documents
- Removes deleted documents
- Rebuilds index with changes

### `index inspect`

View metadata and status of collections.

**Syntax:**
```bash
indexed index inspect [NAME] [OPTIONS]
```

**Arguments:**

- `NAME` (optional): Collection name for detailed inspection (omit to list all)

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--verbose` | `-v` | Show detailed information for all collections | `false` |
| `--json` | | Output as JSON | `false` |

**Examples:**

```bash
# List all collections
indexed index inspect

# Detailed info about all collections
indexed index inspect --verbose

# Inspect specific collection
indexed index inspect my-docs

# JSON output
indexed index inspect my-docs --json
```

**Output:** Shows collection name, type, document count, chunk count, size, and last updated time.

---

### `index remove`

Delete a collection permanently.

**Syntax:**
```bash
indexed index remove <COLLECTION> [OPTIONS]
```

**Arguments:**

- `COLLECTION` (required): Collection name to remove

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--force` | `-f` | Skip confirmation prompt | `false` |
| `--verbose` | `-v` | Enable verbose logging | `false` |
| `--json-logs` | | Output logs as JSON | `false` |
| `--log-level` | | Set logging level | (default) |

**Examples:**

```bash
# Remove with confirmation
indexed index remove old-collection

# Remove without confirmation
indexed index remove old-collection --force
```

**Warning:** This action cannot be undone. All indexed data for the collection is permanently deleted.

## Config Commands

Commands for managing configuration.

### `config init`

Initialize a new workspace with default configuration.

**Syntax:**
```bash
indexed config init [OPTIONS]
```

**What it creates:**
- `config.toml`: Configuration file with defaults
- `.env`: Placeholder for API tokens
- `data/`: Directory for collections

**Examples:**

```bash
# Initialize global workspace
indexed config init

# Initialize local workspace
indexed --local config init
```

---

### `config update`

Interactively modify configuration values.

**Syntax:**
```bash
indexed config update [OPTIONS]
```

**Examples:**

```bash
# Interactive configuration update
indexed config update
```

This command walks you through:
1. Selecting a section (e.g., `sources`, `core`)
2. Choosing a setting to change
3. Entering a new value

---

### `config inspect`

View the current active configuration.

**Syntax:**
```bash
indexed config inspect [SECTION] [OPTIONS]
```

**Arguments:**

- `SECTION` (optional): Section to inspect (sources, core, logging, mcp, performance)

**Options:**

| Flag | Description | Default |
|------|-------------|---------|
| `--json` | Output as JSON | `false` |
| `--show-defaults` | Include default values in output | `false` |

**Examples:**

```bash
# View all configuration
indexed config inspect

# View specific section
indexed config inspect sources

# JSON output
indexed config inspect --json

# Include defaults
indexed config inspect --show-defaults
```

---

### `config set`

Set a specific configuration value directly.

**Syntax:**
```bash
indexed config set <KEY> <VALUE> [OPTIONS]
```

**Arguments:**

- `KEY`: Configuration key (dot-separated path, e.g., `core.v1.indexing.chunk_size`)
- `VALUE`: Value to set

**Examples:**

```bash
# Set chunk size
indexed config set core.v1.indexing.chunk_size 1024

# Set Jira URL
indexed config set sources.jira.url https://company.atlassian.net

# Set storage mode
indexed config set storage.mode local
```

---

### `config delete`

Remove a specific configuration key.

**Syntax:**
```bash
indexed config delete <KEY> [OPTIONS]
```

**Arguments:**

- `KEY`: Configuration key to delete (dot-separated path)

**Examples:**

```bash
# Delete a setting
indexed config delete sources.jira.query
```

---

## MCP Commands

Commands for managing the Model Context Protocol server.

### `mcp run`

Start the MCP server (default: stdio transport).

**Syntax:**
```bash
indexed mcp run [OPTIONS]
```

**Options:**

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--transport` | `-t` | Transport: stdio, http, sse, streamable-http | `stdio` |
| `--host` | | Host for HTTP/SSE transports | `127.0.0.1` |
| `--port` | | Port for HTTP/SSE transports | `8000` |
| `--log-level` | | Logging level | `INFO` |
| `--json-logs` | | Output logs as JSON | `false` |

**Examples:**

```bash
# Start with stdio (for Claude Desktop)
indexed mcp run

# Start HTTP server
indexed mcp run --transport http --port 8000

# Start with custom logging
indexed mcp run --log-level DEBUG
```

---

### `mcp dev`

Run MCP server in development mode with inspector.

**Syntax:**
```bash
indexed mcp dev [OPTIONS]
```

**Examples:**

```bash
# Start dev server with inspector
indexed mcp dev
```

Opens an interactive inspector for testing MCP tools and capabilities.

---

### `mcp inspect`

View capabilities of the MCP server.

**Syntax:**
```bash
indexed mcp inspect [OPTIONS]
```

**Options:**

| Flag | Description | Default |
|------|-------------|---------|
| `--format` | Output format: text, fastmcp, or mcp | (default) |
| `--output` | `-o` | Output file path | (stdout) |
| `--json-output` | | Output as JSON | `false` |

**Examples:**

```bash
# View capabilities
indexed mcp inspect

# JSON output
indexed mcp inspect --json-output

# Save to file
indexed mcp inspect --output capabilities.json
```

**Output:** Lists available tools (`search`, `list_collections`, `collection_status`) and their descriptions.

## Global Flags

These flags apply to almost every command.

| Flag | Short | Description |
|------|-------|-------------|
| `--verbose` | `-v` | Enable detailed (INFO) logging |
| `--json-logs` | | Output logs in structured JSON format |
| `--log-level` | | Set logging level (DEBUG, INFO, WARNING, ERROR) |
| `--local` | | Force operations to use local `./.indexed` workspace |
| `--global` | | Force operations to use global `~/.indexed` workspace |

**Examples:**

```bash
# Use verbose logging
indexed --verbose index create files --name my-docs --path ./docs

# Use local storage mode
indexed --local config inspect

# JSON logs for parsing
indexed --json-logs index search "query"

# Set log level
indexed --log-level DEBUG index update my-docs
```

**Note:** `--local` and `--global` cannot be used together.

### `migrate`

**Syntax:**
```bash
indexed migrate [OPTIONS]
```

**Options:**

| Flag | Description | Default |
|------|-------------|---------|
| `--dry-run` | Show what would be migrated without copying | `false` |

**Examples:**

```bash
# Migrate legacy data
indexed migrate

# Preview migration
indexed migrate --dry-run
```

## Exit Codes

Indexed uses standard exit codes:

- `0`: Success
- `1`: Error (invalid command, file not found, etc.)
- `2`: Configuration error

## Command Aliases

Some commands have alternative names for convenience:

- `indexed knowledge` → same as `indexed index` (hidden)
- `indexed index` → same as `indexed knowledge` (hidden)

Use the documented command names (`indexed index ...`) for clarity.
